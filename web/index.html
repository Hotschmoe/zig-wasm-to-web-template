Feature-Flagged WebAssembly with Dynamic Module Loading

How It Works

This template demonstrates compile-time feature flagging in Zig with dynamic JavaScript module loading.

Build with features:

zig build -Dwebgpu=true -Dwebaudio=true

zig build -Dall=true (enable all features)

zig build (minimal build, no features)

What Happens

1. WASM loads and reports enabled features

2. JavaScript dynamically imports only required glue modules

3. Each feature initializes independently

4. Unused features are eliminated at compile time
